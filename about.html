About this project

This application is meant to allow you to visualize graphs and make searches over them. To do that, first you must load a graph. There are 3 ways to do this:

  • Loading your own graph on the top option from the left panel. The file must have .json or .gexf extension with a valid format. The following is the structure of a correct JSON graph:
    • Array field "links" (mandatory). Each of its elements have:
      • "source" field (mandatory), which is the node whose id is "source" where the link comes from.
      • "target" field (mandatory), which is the node whose id is "target" where the link's arrow points to.
      • "weight" field (optional), which is link's cost. By default, this value will be 1 (but not shown in the graph).
    • Array field "nodes" (optional). Each of its elements can have:
      • "x" field (mandatory if node array used), which is node's x coordinate.
      • "y" field (mandatory if node array used), which is node's y coordinate.
      • "size" field (optional), which is node's graphic size.
      • "color" field (optional), which is node's color. You can specify it with any of CSS's color notations.
    Once you have loaded a valid graph, it will be saved in the server and it will send you back a hash to load the graph in future executions. Don't worry if you didn't copy it when the message appears, as it is also shown in left panel's title.
  • Selecting an example graph from the server.
  • Introducing the hash obtained by the first method.

Once your graph was loaded, you should be able to visualize it if everything went okay. After this, you can do graph-searches after selecting the initial and goal nodes by right-clicking on them. You can select the algorithm to do the search and the way to do it (directly or step by step). You can also change the distribution of the nodes by selecting and applying a layout. You can also change the visualization manually by dragging the nodes or the whole graphic, zooming with the two first lower lens and centering the graph with the third one.

While doing these searchs, nodes are filled with a determined color set that is explained when clicking on the legend in the upper side of the main page. In step-by-step executions, the cost of the expanded node is painter over it. In A*, what is painted over the node is the score function (cost + heuristic estimation), though both euclidean and Manhattan distances are normalized to the default distance of a grid layout (100 pixels).

This project was created by Miguel López Becoña. Its graph-search component is Hipster, which was originally created and is currently maintained by:

Rodriguez-Mier, P., Gonzalez-Sieira, A., Mucientes, M., Lama, M. & Bugarin, A. (2014). Hipster: An Open Source Java Library for Heuristic Search. 9th Iberian Conference on Information Systems and Technologies (CISTI).
